# Contributing

Mahalo for taking the time to help keep these checklists accurate. This is a
small, single-firm-maintained repository, so contributions work a little
differently than a typical open-source project.

## The shortest path — just open a Discussion

For almost all feedback, **opening a thread in
[GitHub Discussions](../../discussions) is the best way to help**.

That includes:

- **Corrections.** If something on a checklist is out of date, wrong, or no
  longer required, please say so — and if you can, point to the current
  source (County page, State statute, MC-12 section, etc.).
- **Clarifying questions.** If an item is confusing, that probably means the
  wording needs fixing. Ask, and we'll rewrite it.
- **Edge cases.** Unusual project types (accretion lots, older County files,
  SSV filings, non-residential work) that the checklists don't cover well.
- **Missing items.** Commonly flagged submission items the checklists ought
  to include.

Discussions are reviewed as part of the quarterly update cycle, and urgent
issues faster than that.

## Pull requests are welcome, with a few notes

If you'd like to open a pull request:

1. **For content fixes** (changes to either checklist), please edit the
   generator source in `_generators/`, not the generated HTML or PDF
   directly. The HTML and PDF are built from the Python source and any
   hand-edits will be overwritten the next time the generator runs.
2. **For the landing page** (`index.html`), feel free to edit it directly.
3. Please include a short description of **what rule or source** supports
   your change so it can be added to `CHANGELOG.md`.
4. All edits, including PRs, land through the same quarterly review. We
   can't always merge immediately.

## What's not a good fit here

- **Specific project advice.** We can't give binding guidance on your
  particular project through a public forum — please talk to a licensed
  professional or the permitting agency directly.
- **Disputes with the County or State.** These checklists are a guide,
  not a place to take issue with agency policy.

## Code of conduct

Please be kind. Most people reading these checklists are trying to build a
home, a small business, or a project they care about. Treat other
contributors and readers the way you would treat a neighbor who walked into
your studio with a question.
